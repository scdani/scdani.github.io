The Problem

Issue Reporting on the Shop Floor is Broken

🗂️
Unstructured reporting
Teams, phone, handwritten notes — asset IDs and error codes routinely missing.
🔁
10–15 min of follow-up per ticket
Triage asks 5+ questions every time. Context is already gone by then.
📚
Knowledge buried
Guides live in PDFs. Tribal knowledge stays in people's heads.
Today's reality
✗ Reports missing asset ID, location, error codes, photos
✗ Priority assigned inconsistently — depends on who's on shift
✗ Free-text chaos — impossible to trend or analyse
✗ Junior technicians under-report or over-escalate
What better looks like
✓ Issue captured in under 60 seconds — at the asset, on the tablet
✓ AI asks only the missing critical questions — not a long form
✓ Consistent structured data every time — ready for triage
✓ Relevant guidance surfaced in context — not buried in PDFs

The Solution

Field Assist — How It Works

01
Select asset & describe the issue
Type or dictate. Photo optional. Safety & production impact in one tap. Under 60 seconds.
02
AI Assist — structure + follow-up
Summary generated. Missing fields detected. 3–5 targeted questions asked. Category, priority + rationale suggested.
03
Confirm & submit
Technician reviews, edits any field, submits. Full structured report lands in backoffice queue instantly.
Design principles
✓ 2–3 screens max for technicians
✓ Touch targets ≥ 48px — works with gloves
✓ High contrast for industrial lighting
✓ Drafts stored locally when offline, sync on reconnect
✓ AI degrades gracefully — manual entry always works
User roles
→ Technician — create report, answer AI questions, submit
→ Supervisor / Backoffice — review queue, reclassify, assign, convert to work order
→ IT / Security — data boundaries, auditability, retention policy

AI Assist

Responsible AI — Embedded, Not Intrusive

📝
Summarise the observation
Structured summary from free-text or voice. Fully editable.
❓
Ask only what's missing
Up to 5 targeted follow-up questions. "Unknown" always an option.
🏷️
Category + priority with rationale
P1/P2/P3 suggestion with a short "why". Technician confirms or overrides.
🔍
Safe recommended checks
Up to 3 non-invasive checks from approved knowledge base. Never unsafe instructions.
🛡️AI suggestions are never instructions. Human confirmation required on every field.
⚡ AI Assist — live example
Technician: "Motor on Line 3 overheats after 10 min, smell of burning"
What I understood: Overheating motor on Line 3, onset after ~10 min of operation, accompanied by burning smell. Emergency stop status unknown. Specific motor and load not yet recorded.
Missing 3 critical fields — asking:
1 Which motor exactly? (asset ID or label)
2 Did it trigger an emergency stop? (Yes / No)
3 Any error code on the control panel?
P2 — Production impact Overheating
Why: production impacted, no confirmed safety risk, no workaround noted.
✓ Human confirmation required before submitting

MVP Scope & Pilot

Small Enough to Test Fast. Meaningful Enough to Prove Value.

In scope — MVP
✓ Tablet-first Report Issue flow (2–3 screens, glove-friendly)
✓ Voice-to-text + photo attachments
✓ AI Assist: summary, follow-ups, category, priority, safe checks
✓ Human-confirmed before submit
✓ Backoffice review queue with filters & triage
✓ Offline-tolerant — syncs on reconnect
Out of scope — MVP
– Full ITSM / work order system write-back (queue export first)
– Fully offline AI reasoning (graceful degradation to manual)
– Enterprise SSO / advanced RBAC (pilot-friendly login first)
Success targets
Triage follow-up messages−50%
Reports with asset ID>95%
AI summary acceptance rate>70%
Average report capture time<60s
Pilot plan
PRD + User stories
Prototype 2–3 screens
MVP build
2-week pilot (5 techs, 1 zone)
Evaluate & expand
